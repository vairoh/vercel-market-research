You are working on my React page (ResearchPage.tsx). In the “Most used keywords” input, each selected keyword renders as a pill/chip (e.g. “#finops ×”). The pill is stretching too wide, taking up most of the row, instead of wrapping tightly around its content.

Root cause: the chips are flex items inside a flex-wrap container, and their sizing contract is not defined. In flex layout, relying on width: fit-content is not robust. Some CSS (global reset or inherited styles) may be causing the chip element to behave like a block or to grow (flex-grow / width: 100%), which makes it stretch. The professional fix is to define a proper “chip” style contract that is content-sized and non-growing, then apply it consistently via a class or component.

Implement this fix:
1) Create a small scoped CSS file (CSS module preferred) for the keyword area, and move chip styling there (avoid inline width hacks).
2) Ensure the chip has:
   - display: inline-flex
   - flex: 0 0 auto (critical so it never grows)
   - white-space: nowrap (keeps “#kw ×” on one line)
   - max-width: 100% (prevents overflow on very long keywords)
3) Ensure the parent container remains:
   - display: flex; flex-wrap: wrap; gap: ...
4) Update ResearchPage.tsx to use className for the container and chip.
